Como usar o Atlas

Guia prático para criar, organizar e publicar conteúdos sem se perder em pastas, arquivos e caminhos.

Regra principal: o Atlas começa em Criar Conteúdo. Abrir projeto só aparece quando for preciso definir ou trocar a raiz.

1. Criar conteúdo

Abra o Atlas em:

cms/index.html

A primeira ação é preencher:

  • Nome do conteúdo: título do artigo, pensador, livro etc.
  • Tipo: Artigo, Pensador, Livro, Conceito etc.

Depois clique em:

Criar Conteúdo

2. Primeira utilização

Se for a primeira vez, o Atlas pedirá a raiz do projeto. Selecione a pasta principal:

Mente-Crua/

Não entre em modules, cms ou assets.

3. Estrutura criada

O Atlas cria automaticamente:

modules/
└── artigos/
    └── quando-o-pequeno-principe-morre-dentro-de-nos/
        ├── index.html
        ├── data.json
        └── img/

4. Colar HTML

Cole o conteúdo completo no campo Conteúdo HTML.

<article>
  <section>
    <h2>Título</h2>
    <p>Texto do artigo...</p>
    <img src="img/imagem-01.webp" alt="Descrição">
  </section>
</article>

O HTML não deve conter header, footer, GTM, menu, busca ou compartilhamento. Isso vem do template.

5. Preencher SEO

  • Título SEO
  • Descrição SEO
  • Categoria
  • Tags
  • Relacionamentos por enquanto em texto simples

6. Enviar imagens

Clique em Enviar Imagens. Todas irão para:

modules/artigos/slug-do-artigo/img/

Padrão recomendado:

thumb.webp
imagem-01.webp
imagem-02.webp

7. Definir thumb

Clique em Usar como Thumb. O campo deve ficar assim:

img/thumb.webp

8. Publicar

Clique em Publicar. O Atlas deve salvar o conteúdo, gerar a página e atualizar a busca/Home quando o Builder estiver completo.

Se o Atlas pedir pasta na hora de publicar, é bug. Ele já deveria conhecer a raiz.

9. Testar

  • Header correto.
  • Rodapé correto.
  • Compartilhamento com WhatsApp, Facebook e Copiar Link.
  • Thumb aparecendo.
  • Busca encontrando o conteúdo.
  • GTM presente no template final.

10. Commit

git add .
git commit -m "feat: publica novo artigo"
git push

Abrir conteúdo existente

Use Abrir Conteúdo para editar algo já criado. Nesta versão ele ainda abre o data.json, mas a interface final mostrará uma lista pesquisável de conteúdos.
